# 量子纠缠交互实验室

## 项目简介

量子纠缠交互实验室是一个面向双光子纠缠实验教学、贝尔不等式检验展示与 CHSH 关联测量理解的多模态交互系统。项目融合三维建模、网页端三维渲染、沉浸式展示、WebAR 图像识别与实验数据驱动模型，使观众能够从空间结构、沉浸视角、真实环境叠加和参数调节四个层面理解量子纠缠实验。

系统围绕 405nm 泵浦光、BBO 晶体、自发参量下转换、偏振分析、单光子探测、符合计数与 CHSH 的 S 值展开。网页端所有主要入口统一部署在同一在线地址下，适合课堂演示、作品答辩、物理实验创新竞赛展示和移动端扫码体验。

## 功能模块

### 3D 展示模式

3D 展示模式用于自由观察量子纠缠实验装置。观众可以旋转、缩放并查看关键器件说明，理解泵浦光、BBO 晶体、偏振分析器与符合计数模块之间的空间关系。

### VR 沉浸模式

VR 沉浸模式以第一人称视角进入实验场景，展示纠缠光子产生、路径传播、偏振分析与符合计数过程。该模式适合在手机横屏或桌面端进行沉浸式讲解。

### AR 扫描模式

AR 扫描模式通过 WebAR 图像识别，将动态量子纠缠实验装置叠加到真实环境中。观众允许摄像头权限并对准识别图后，即可在手机屏幕中观察实验装置、光路和关键结构。

### 双光子纠缠 Demo

双光子纠缠 Demo 提供参数调节和计数观察功能。观众可以改变泵浦强度、偏振角、补偿状态、背景噪声、滤光片带宽和符合窗口，实时观察单路计数、符合计数、关联函数和 CHSH 的 S 值变化。

### 3D Demo

3D Demo 以 Three.js 三维交互方式补充展示实验光路结构和器件位置，帮助观众从三维空间中理解双光子纠缠实验布局。

## 技术栈

- Blender：实验装置建模与三维资源制作
- GLB / glTF：三维模型资源发布格式
- Three.js：网页端三维渲染、相机控制、光效与动画
- WebAR：图像识别与现实叠加展示
- JavaScript / React / Vite：交互页面、Demo 集成与构建
- GitHub Pages：在线访问与展示发布
- 实验数据驱动模型：单路计数、符合计数、关联函数与 CHSH 结果预测

## 物理背景

双光子纠缠实验通常通过 BBO 晶体中的自发参量下转换产生纠缠光子对。实验中，泵浦光进入非线性晶体后产生相关光子，随后通过偏振分析器和单光子探测器记录单路计数与符合计数。

CHSH 不等式是检验贝尔不等式的重要形式。经典局域实在论给出上界 `S <= 2`，而量子纠缠态在合适的偏振角设置下可以得到 `S > 2` 的结果。本项目通过可视化光路与可调参数的计数展示，帮助观众理解这一物理意义。

## 实验数据驱动模型

系统使用基于实测计数数据校准的 CHSH 教学近似模型。模型以实验测得的单路计数、符合计数和 CHSH 角度组合数据为基础，结合偏振相关函数、探测效率、背景噪声和偶然符合等因素，生成网页端实时展示所需的计数和 S 值。

模型输入包括：

- 泵浦强度
- 偏振角
- 补偿状态
- 背景噪声
- 符合窗口
- 探测效率

模型输出包括：

- 单路计数
- 符合计数
- E 值
- S 值
- 可见度
- 偶然符合

该模型用于教学演示和交互理解，不替代精密实验数据分析。它的目标是让参数变化、符合计数变化和 CHSH 结果之间保持清晰、连续且符合物理规律的关系。

## 使用方法

1. 打开在线地址或本地预览地址。
2. 在首页选择需要展示的模式。
3. 使用 3D 展示模式观察实验装置空间结构。
4. 使用 VR 沉浸模式从第一人称视角观察实验过程。
5. 使用 AR 扫描模式对准识别图，体验真实环境叠加展示。
6. 使用双光子纠缠 Demo 调节参数并观察计数和 S 值变化。
7. 使用 3D Demo 补充理解实验光路和器件布局。

## 本地运行

安装依赖：

```bash
npm.cmd install
```

启动本地开发服务：

```bash
npm.cmd run dev -- --host 0.0.0.0
```

生成正式构建：

```bash
npm.cmd run build
```

本地预览正式构建：

```bash
npm.cmd run preview -- --host 0.0.0.0
```

## 发布说明

项目使用 Vite 构建，站点路径配置为 `/Quantum-vr/`，与 GitHub Pages 在线地址保持一致。发布前建议先生成展示二维码并完成正式构建：

```bash
npm.cmd run qr -- --url "https://ysomething.github.io/Quantum-vr/"
npm.cmd run build
```

发布到 GitHub Pages 时，请在完成正式构建后执行项目中配置的发布脚本，并在在线地址中检查首页与五个入口页面。
